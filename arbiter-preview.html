Previewing Arbiter 0.1 beta.

Arbiter is an experimental hosted coding route for Orrery subscribers. It is meant to help coding agents produce work with clearer evidence, usage receipts, and review boundaries. It is not a public model-weights release, not a benchmark claim, and not a guarantee of correctness.

Arbiter is a premium Orrery route for coding tasks where evidence matters. The public promise is intentionally simple: ask Orrery to run work, review what happened, see the usage it consumed, and decide whether to trust the result.

What subscribers see

Inside Orrery, Arbiter appears as a hosted model route named arbiter-0.1-beta. It can be selected from Nexus alongside DeepSeek API, Doubleword, BYOK, and local routes. Subscribers see usage meters, run receipts, and proof/boundary summaries where the app supports them.

The route is designed for agentic coding workflows: planning, editing, follow-up fixes, and review. It does not replace your own testing, code review, security review, or backups.

Beta boundary: Arbiter may be slow, unavailable, changed, or paused while the beta is measured. Results vary by task, context, tools, and provider availability.

What is not public

Arbiter is not an open-weights release. Private weights, serving details, internal prompts, routing internals, and operations details are not part of the public beta page. The subscriber-facing product surface is the hosted route inside Orrery.

Evidence boundary

Current internal beta evidence includes a 92/100 frozen local projection on the first 100 SWE-bench Verified tasks. That is not an official leaderboard score, not a live-service guarantee, and not a promise about any subscriber result. It is an internal release signal, and public claims stay conservative until larger matched evaluations are complete.

Availability

Arbiter access is planned for Orrery Pro, Max, and Ultra subscribers through hosted credits. The exact capacity, latency, quotas, model mix, and availability may change during the beta.

Use safely

Treat Arbiter output like any other AI-generated code: inspect it, run tests, review security implications, and keep backups. Orrery can help organize evidence, but you remain responsible for what you accept, run, and ship.
