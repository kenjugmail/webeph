Orrery Nexus

One console for workspaces, agents, routes, and receipts.

Nexus is the daily control surface: recent workspaces on the left, the active run in the center, and files, model route, agent details, proof, and usage on the right. It is built to make the common path obvious without hiding the review trail.

Daily run loop

Send work from one place, then decide what to trust.

The default flow is deliberately small: open a workspace, choose a route, send a prompt, watch the visible progress stream, then inspect summary, changed-file metadata, usage, and safety receipts. Deeper controls live in Nexus Settings.

⌘

Open folder

Start from the project folder you want Orrery to work in. Empty folders can be initialized as new workspaces.

◉

Pick a route

Use local, BYOK, or subscription-hosted routes. Arbiter appears as a beta hosted coding route for eligible subscribers.

▷

Run an agent

Nexus prepares workspace metadata, runs a safety check, and starts a registered agent when the route is ready.

◇

Review receipts

Use proof, boundary, usage, and final summaries to decide whether to keep, revise, or discard the result.

Safety and beta boundaries

Helpful automation, not blind trust.

Orrery can make mistakes, and generated code may be insecure, incomplete, or unsuitable for your project. The product is designed to keep work reviewable: scope checks, permission gates, model-route labels, proof summaries, and usage receipts stay visible where supported.

▦Review before shippingYou are responsible for accepting, running, and deploying generated code.
◈No guaranteed outcomePerformance depends on task, context, models, tools, and provider availability.
⬡Keep backupsUse source control and inspect changes before relying on them.
◐Cloud variesBeta routes, quotas, latency, and providers may change or pause.
